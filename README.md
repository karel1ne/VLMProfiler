# Профилировка LLM и VLLM

## Команда
- Егор (@karel1ne)
- Никита (@epicfiredragon)

## Описание проекта
Изучение масштабируемости VLLM. Анализ узких мест, поиск путей оптимизации latency и энергопотребления. Основная цель — выявить части моделей, требующие наибольшей оптимизации, и определить наиболее оптимальные модели по соотношению скорость/качество/энергопотребление.

## Оборудование

## Датасеты
- ScienceQA (mini-val subset)
- TextVQA (val subset)
- COCO Caption Val (subset)

## Метрики
- Latency (мс)
- FLOPS
- Энергопотребление
- Качество WER

## Оси масштабирования
- Разрешение изображения: 224 → 336 → 448 → 512 (если поддерживается)
- Длина текстового запроса: 10 → 50 → 100 → 200 токенов
- Тип устройства: CPU vs GPU (Quadro RTX 4000 8GB / RTX A4000 16GB)
- Оптимизации: FP16, Quantization (4/8 bit), torch.compile, FlashAttention-2
- Batch size: 1 → 2 → 4 → 8 (если позволяет память)
